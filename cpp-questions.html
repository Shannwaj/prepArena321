C++ Interview Questions

Master C++ concepts for technical interviews and coding challenges

1. What are the differences between C and C++?

Key differences include:

  • C++ supports object-oriented programming with classes and objects, while C is procedural
  • C++ supports function overloading, C doesn't
  • C++ has exception handling, C doesn't
  • C++ supports references, C doesn't
  • C++ has the Standard Template Library (STL), C doesn't
  • C++ supports operator overloading, C doesn't
  • C++ has namespace feature, C doesn't
  • C++ supports function templates and class templates, C doesn't

2. What is the difference between new/delete and malloc/free in C++?

Main differences:

  • new/delete are operators that call constructors/destructors, while malloc/free are functions that don't
  • new returns exact data type, while malloc returns void*
  • new can be overloaded, malloc cannot
  • new throws exceptions on failure, malloc returns NULL
  • new allocates memory from the free store, malloc allocates from the heap

// Using new/delete
int* p1 = new int(10);
delete p1;

// Using malloc/free
int* p2 = (int*)malloc(sizeof(int));
*p2 = 10;
free(p2);

3. Explain virtual functions and polymorphism in C++

Virtual functions enable polymorphism in C++:

  • Declared with 'virtual' keyword in base class
  • Allow derived classes to override the base implementation
  • Resolved at runtime (dynamic binding) based on the object's actual type
  • Implemented using virtual function tables (vtables)
  • Enable "late binding" through pointers or references

class Base {
public:
    virtual void show() { cout << "Base class"; }
    // Pure virtual function (abstract)
    virtual void pure() = 0;
};

class Derived : public Base {
public:
    void show() override { cout << "Derived class"; }
    void pure() override { cout << "Implemented"; }
};

// Usage:
Base* obj = new Derived();
obj->show(); // Outputs: "Derived class"

4. What is the difference between a struct and a class in C++?

The only technical difference is the default access level:

  • In a struct, members are public by default
  • In a class, members are private by default
  • Both can have member functions, constructors, destructors, etc.
  • Both can inherit from other classes/structs
  • Both can use access specifiers (public, private, protected)

Convention suggests:

  • Use struct for passive objects with public data
  • Use class for active objects with private data and public interfaces

5. Explain the different types of inheritance in C++

C++ supports five types of inheritance:

  • Single inheritance: A class inherits from one base class
  • Multiple inheritance: A class inherits from multiple base classes
  • Hierarchical inheritance: Multiple classes inherit from a single base class
  • Multilevel inheritance: A class inherits from a derived class
  • Hybrid inheritance: Combination of multiple and hierarchical inheritance

// Single inheritance
class Derived : public Base { };

// Multiple inheritance
class Derived : public Base1, public Base2 { };

// Multilevel inheritance
class C : public B { }; // where B : public A { };

6. What is the diamond problem in C++ and how is it resolved?

The diamond problem occurs in multiple inheritance when a class inherits from two classes that inherit from a common base class:


class A { public: int x; };
class B : public A { };
class C : public A { };
class D : public B, public C { }; // Diamond problem!

The problem: Class D has two copies of all members from class A.

Solution: Use virtual inheritance:


class A { public: int x; };
class B : virtual public A { };
class C : virtual public A { };
class D : public B, public C { }; // Resolved

Virtual inheritance ensures only one copy of the base class exists in the inheritance hierarchy.

7. What are smart pointers and how do they work?

Smart pointers are objects that act like pointers but provide automatic memory management:

  • unique_ptr: Exclusive ownership model; cannot be copied
  • shared_ptr: Shared ownership model; uses reference counting
  • weak_ptr: Non-owning reference to shared_ptr; helps break circular references

#include 

// unique_ptr
std::unique_ptr u_ptr = std::make_unique(10);

// shared_ptr
std::shared_ptr s_ptr1 = std::make_shared(20);
std::shared_ptr s_ptr2 = s_ptr1; // Reference count = 2

// weak_ptr
std::weak_ptr w_ptr = s_ptr1;

Benefits include:

  • Automatic memory deallocation when no longer needed
  • Prevention of memory leaks
  • Exception safety

8. Explain move semantics and perfect forwarding in C++11

Move Semantics allows transferring resources (like memory ownership) from one object to another, avoiding unnecessary copying:

  • Utilizes rvalue references (&&)
  • Move constructor and move assignment operator
  • std::move() converts an object to an rvalue reference

// Move constructor
MyClass(MyClass&& other) noexcept 
    : data(other.data) {
    other.data = nullptr; // Transfer ownership
}

// Usage
MyClass obj1;
MyClass obj2 = std::move(obj1); // Move from obj1 to obj2

Perfect Forwarding allows passing arguments while preserving their value category (lvalue/rvalue):


template
std::unique_ptr make_unique(Args&&... args) {
    return std::unique_ptr(new T(std::forward(args)...));
}

std::forward ensures arguments are forwarded with their original value category intact.

9. What are lambda expressions in C++?

Lambda expressions are anonymous functions that can be defined inline:


// Basic lambda syntax
[capture](parameters) -> return_type { body }

// Example
auto add = [](int a, int b) -> int { return a + b; };
int sum = add(5, 3); // sum = 8

// Capture variables by value
int x = 10;
auto incrementBy = [x](int y) { return x + y; };

// Capture variables by reference
auto modify = [&x]() { x += 5; };

// Capture all variables
auto all = [=]() { /* use all variables by value */ };
auto allRef = [&]() { /* use all variables by reference */ };

Lambdas are useful for:

  • Short, one-time use functions
  • Function arguments (e.g., for STL algorithms)
  • Callbacks

10. What is RAII in C++ and why is it important?

RAII (Resource Acquisition Is Initialization) is a C++ programming idiom:

  • Resources are acquired during object construction
  • Resources are released during object destruction
  • Resources include memory, file handles, locks, network connections, etc.

class FileHandler {
private:
    FILE* file;
public:
    // Constructor acquires resource
    FileHandler(const char* filename) {
        file = fopen(filename, "r");
        if (!file) throw std::runtime_error("File open failed");
    }
    
    // Destructor releases resource
    ~FileHandler() {
        if (file) fclose(file);
    }
    
    // Prevent copying
    FileHandler(const FileHandler&) = delete;
    FileHandler& operator=(const FileHandler&) = delete;
};

// Usage
{
    FileHandler fh("example.txt"); // File opened
    // Use file...
} // File automatically closed when fh goes out of scope

Benefits of RAII:

  • Automatic resource management
  • Exception safety
  • Prevents resource leaks
  • Cleaner code with fewer try-catch blocks

11. What is a friend function and friend class in C++?

Friend functions and classes are given special access to private and protected members:

  • A friend function is not a member function but can access private/protected members
  • A friend class can access private/protected members of the class that declared it as a friend

class MyClass {
private:
    int data = 10;
    
    // Friend function declaration
    friend void display(MyClass obj);
    
    // Friend class declaration
    friend class Helper;
};

// Friend function definition
void display(MyClass obj) {
    // Can access private members
    std::cout << obj.data << std::endl;
}

// Friend class
class Helper {
public:
    void modify(MyClass& obj) {
        // Can access private members
        obj.data = 20;
    }
};

Important considerations:

  • Friendship is not transitive (if A is friend of B, and B is friend of C, A is not automatically friend of C)
  • Friendship is not inherited
  • Friendship breaks encapsulation, so use sparingly

12. Explain function overloading and operator overloading in C++

Function Overloading allows multiple functions with the same name but different parameters:


void print(int x) {
    std::cout << "Integer: " << x << std::endl;
}

void print(double x) {
    std::cout << "Double: " << x << std::endl;
}

void print(std::string s) {
    std::cout << "String: " << s << std::endl;
}

// Usage
print(5);       // Calls print(int)
print(3.14);    // Calls print(double)
print("Hello"); // Calls print(std::string)

Operator Overloading allows redefining operators for custom types:


class Complex {
private:
    double real, imag;
    
public:
    Complex(double r, double i) : real(r), imag(i) {}
    
    // Overload + operator
    Complex operator+(const Complex& other) const {
        return Complex(real + other.real, imag + other.imag);
    }
    
    // Overload << operator as a friend function
    friend std::ostream& operator<<(std::ostream& os, const Complex& c);
};

std::ostream& operator<<(std::ostream& os, const Complex& c) {
    os << c.real << " + " << c.imag << "i";
    return os;
}

// Usage
Complex a(1.0, 2.0);
Complex b(3.0, 4.0);
Complex c = a + b;      // Uses operator+
std::cout << c << std::endl; // Uses operator<<
