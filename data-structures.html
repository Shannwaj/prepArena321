Data Structure Categories

Easy Arrays

Maximum Subarray

Given an integer array nums, find the contiguous subarray (containing at least one number) which has the largest sum and return its sum.

Example:


Input: nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]
Output: 6
Explanation: [4, -1, 2, 1] has the largest sum = 6

Solution:


def max_subarray(nums):
    if not nums:
        return 0
        
    max_sum = current_sum = nums[0]
    
    for num in nums[1:]:
        current_sum = max(num, current_sum + num)
        max_sum = max(max_sum, current_sum)
        
    return max_sum

# Time Complexity: O(n)
# Space Complexity: O(1)

Explanation:

This is Kadane's algorithm for finding the maximum subarray sum:

  • Initialize two variables: current_sum (to track current subarray sum) and max_sum (to track the maximum sum found so far)
  • For each element, decide whether to extend the previous subarray or start a new subarray from the current element
  • Update the maximum sum if the current subarray sum is larger
Medium Arrays

Product of Array Except Self

Given an integer array nums, return an array answer such that answer[i] is equal to the product of all the elements of nums except nums[i].

The algorithm should run in O(n) time and without using the division operation.

Example:


Input: nums = [1, 2, 3, 4]
Output: [24, 12, 8, 6]
Explanation: 
answer[0] = 2*3*4 = 24
answer[1] = 1*3*4 = 12
answer[2] = 1*2*4 = 8
answer[3] = 1*2*3 = 6

Solution:


def product_except_self(nums):
    n = len(nums)
    answer = [1] * n
    
    # Calculate products of all elements to the left
    left_product = 1
    for i in range(n):
        answer[i] = left_product
        left_product *= nums[i]
    
    # Multiply by products of all elements to the right
    right_product = 1
    for i in range(n-1, -1, -1):
        answer[i] *= right_product
        right_product *= nums[i]
    
    return answer

# Time Complexity: O(n)
# Space Complexity: O(1) excluding the output array

Explanation:

  • First pass: For each position, compute the product of all elements to its left
  • Second pass: For each position, multiply by the product of all elements to its right
  • This gives us the product of all elements except the one at the current position
  • We don't use division, making it work even with zero values in the array
Medium Linked Lists

Detect Cycle in a Linked List

Given head, the head of a linked list, determine if the linked list has a cycle in it.

There is a cycle in a linked list if there is some node in the list that can be reached again by continuously following the next pointer.

Example:


Input: head = [3, 2, 0, -4] with a cycle where -4 points back to 2
Output: true
Explanation: There is a cycle in the linked list, where the last node points to the second node.

Solution:


def has_cycle(head):
    if not head or not head.next:
        return False
    
    slow = head
    fast = head.next
    
    while slow != fast:
        if not fast or not fast.next:
            return False
        slow = slow.next
        fast = fast.next.next
    
    return True

# Time Complexity: O(n)
# Space Complexity: O(1)

Explanation:

  • This uses Floyd's Cycle-Finding Algorithm (the "tortoise and hare" approach)
  • We use two pointers moving at different speeds (slow moves 1 step, fast moves 2 steps)
  • If there's a cycle, the fast pointer will eventually catch up to the slow pointer
  • If there's no cycle, the fast pointer will reach the end of the list
Medium Trees

Binary Tree Level Order Traversal

Given the root of a binary tree, return the level order traversal of its nodes' values (i.e., from left to right, level by level).

Example:


Input: root = [3, 9, 20, null, null, 15, 7]
    3
   / \
  9  20
    /  \
   15   7
Output: [[3], [9, 20], [15, 7]]

Solution:


from collections import deque

def level_order(root):
    if not root:
        return []
    
    result = []
    queue = deque([root])
    
    while queue:
        level_size = len(queue)
        level = []
        
        for _ in range(level_size):
            node = queue.popleft()
            level.append(node.val)
            
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        
        result.append(level)
    
    return result

# Time Complexity: O(n)
# Space Complexity: O(n)

Explanation:

  • Use a queue to perform breadth-first search (BFS)
  • For each level, process all nodes currently in the queue
  • Add all children to the queue for the next level
  • Keep track of each level's nodes in a separate list
Easy Hash Table

Valid Anagram

Given two strings s and t, return true if t is an anagram of s, and false otherwise.

An anagram is a word or phrase formed by rearranging the letters of a different word or phrase, typically using all the original letters exactly once.

Example:


Input: s = "anagram", t = "nagaram"
Output: true

Input: s = "rat", t = "car"
Output: false

Solution:


def is_anagram(s, t):
    if len(s) != len(t):
        return False
    
    char_count = {}
    
    # Count characters in string s
    for char in s:
        char_count[char] = char_count.get(char, 0) + 1
    
    # Decrement counts for characters in string t
    for char in t:
        if char not in char_count or char_count[char] == 0:
            return False
        char_count[char] -= 1
    
    return True

# Time Complexity: O(n)
# Space Complexity: O(k) where k is the size of the character set

Explanation:

  • Use a hash map to count the occurrences of each character in the first string
  • Decrement the count for each character in the second string
  • If a character in the second string doesn't exist in the map or has count 0, it's not an anagram
  • Alternative approach: Sort both strings and compare (O(n log n) time)
Hard Advanced

LRU Cache Implementation

Design and implement a data structure for Least Recently Used (LRU) cache. It should support the following operations: get and put.

get(key) - Get the value (will always be positive) of the key if the key exists in the cache, otherwise return -1.

put(key, value) - Set or insert the value if the key is not already present. When the cache reaches its capacity, it should invalidate the least recently used item before inserting a new item.

Example:


LRUCache cache = new LRUCache(2); // capacity = 2
cache.put(1, 1);
cache.put(2, 2);
cache.get(1);       // returns 1
cache.put(3, 3);    // evicts key 2
cache.get(2);       // returns -1 (not found)
cache.put(4, 4);    // evicts key 1
cache.get(1);       // returns -1 (not found)
cache.get(3);       // returns 3
cache.get(4);       // returns 4

Solution:


class DLinkedNode:
    def __init__(self, key=0, value=0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None

class LRUCache:
    def __init__(self, capacity):
        self.cache = {}
        self.size = 0
        self.capacity = capacity
        # Use dummy head and tail nodes
        self.head = DLinkedNode()
        self.tail = DLinkedNode()
        self.head.next = self.tail
        self.tail.prev = self.head

    def _add_node(self, node):
        # Always add the new node right after head
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node

    def _remove_node(self, node):
        # Remove an existing node from the linked list
        prev = node.prev
        new = node.next
        prev.next = new
        new.prev = prev

    def _move_to_head(self, node):
        # Move a node to the front (most recently used)
        self._remove_node(node)
        self._add_node(node)

    def _pop_tail(self):
        # Pop the current tail
        res = self.tail.prev
        self._remove_node(res)
        return res

    def get(self, key):
        node = self.cache.get(key, None)
        if not node:
            return -1
        # Move the accessed node to the head (mark as recently used)
        self._move_to_head(node)
        return node.value

    def put(self, key, value):
        node = self.cache.get(key)
        if not node:
            newNode = DLinkedNode(key, value)
            self.cache[key] = newNode
            self._add_node(newNode)
            self.size += 1
            if self.size > self.capacity:
                # Pop the least recently used item
                tail = self._pop_tail()
                del self.cache[tail.key]
                self.size -= 1
        else:
            # Update the value and move to head
            node.value = value
            self._move_to_head(node)

# Time Complexity: O(1) for both put and get operations
# Space Complexity: O(capacity)

Explanation:

  • Combine a hash map with a doubly linked list to achieve O(1) operations
  • The hash map provides O(1) lookup by key
  • The doubly linked list maintains the order of usage (most recent at front)
  • When capacity is reached, remove the least recently used element (tail of list)
  • Every time an element is accessed, move it to the front (head) of the list

Programming Language Questions

Master language-specific concepts for your technical interviews

What are the differences between C and C++?

Key differences include:

  • C++ supports object-oriented programming with classes and objects, while C is procedural
  • C++ supports function overloading, C doesn't
  • C++ has exception handling, C doesn't
  • C++ supports references, C doesn't
  • C++ has the Standard Template Library (STL), C doesn't

What is the difference between new/delete and malloc/free in C++?

Main differences:

  • new/delete are operators that call constructors/destructors, while malloc/free are functions that don't
  • new returns exact data type, while malloc returns void*
  • new can be overloaded, malloc cannot
  • new throws exceptions on failure, malloc returns NULL

Explain virtual functions in C++

Virtual functions enable polymorphism in C++:

  • Declared with 'virtual' keyword in base class
  • Allow derived classes to override the base implementation
  • Resolved at runtime (dynamic binding) based on the object's actual type
  • Implemented using virtual function tables (vtables)
  • Enable "late binding" through pointers or references

class Base {
public:
    virtual void show() { cout << "Base class"; }
};

class Derived : public Base {
public:
    void show() override { cout << "Derived class"; }
};

// Usage:
Base* obj = new Derived();
obj->show(); // Outputs: "Derived class"

What are Python decorators?

Decorators are a powerful feature in Python that allow you to modify the behavior of functions or classes:

  • They use the @syntax followed by the decorator name above a function definition
  • Decorators are functions that take another function as an argument and return a new function
  • They're used for cross-cutting concerns like logging, timing, authentication, etc.

def timer(func):
    def wrapper(*args, **kwargs):
        import time
        start = time.time()
        result = func(*args, **kwargs)
        print(f"Function {func.__name__} took {time.time() - start:.2f} seconds")
        return result
    return wrapper

@timer
def slow_function():
    import time
    time.sleep(1)
    
# When called, it will print execution time
slow_function()

What is the difference between lists and tuples in Python?

Key differences:

  • Mutability: Lists are mutable (can be changed), tuples are immutable
  • Syntax: Lists use square brackets [], tuples use parentheses ()
  • Performance: Tuples are slightly faster than lists
  • Memory: Tuples use less memory than lists
  • Use cases: Lists for homogeneous data, tuples for heterogeneous data
  • Hashability: Tuples can be used as dictionary keys, lists cannot

Explain Python's Global Interpreter Lock (GIL)

The Global Interpreter Lock (GIL) is a mutex that protects access to Python objects:

  • It allows only one thread to execute Python bytecode at a time
  • Prevents true multi-threading in CPU-bound Python programs
  • IO-bound programs can still benefit from threading despite the GIL
  • Alternatives for CPU-bound tasks: multiprocessing, asyncio, or C extensions
  • The GIL simplifies memory management but limits multi-core utilization
